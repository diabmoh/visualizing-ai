Sources and simplifications

This exhibit compresses one possible single-agent architecture into a deterministic visual story. All places, weather, routes, schemas, prices, tool responses, decision summaries, state changes, and receipts are fictional and illustrative. No live model, API, booking system, or hidden reasoning process is used.

Real agents vary in architecture. Tools may be APIs, functions, databases, code environments, or user interfaces. Memory can be assembled in several ways, and some systems use fixed workflows alongside model-directed decisions. Reliability and appropriate autonomy depend on the task, model, tools, policies, environment, and oversight.

  • Building effective agents for the distinction between fixed workflows and model-directed agents
  • ReAct: Synergizing Reasoning and Acting in Language Models for interleaved actions and observations
  • Toolformer for language-model tool use
  • OpenAI Responses API reference as one implementation example of structured tool requests
  • NIST: Tool Use in Agent Systems for constrained read and write access
  • τ-bench for policy-following, task-state, and repeat-reliability evaluation
  • InjecAgent for indirect prompt-injection risks in tool-integrated agents
